Python形参中*的作用(非*args**kwargs)

2021-05-19

因业务需求， 阅读了python多线程库threading源码。发现__init__函数是这样定义的

class Thread:

    def __init__(self, group=None, target=None, name=None,
                 args=(), kwargs=None, *, daemon=None):

其中 * 瞬间打开了我新世界大门，因为按理说python不支持特殊字符作为变量名。

所有中文搜索引擎没有发现相关解释。 不得已去 栈溢出 网站看看外国大佬怎么说， 很幸运找到了解释。

根据PEP3102，* 的基本意义为， 在这个位置之后的参数必须用关键字参数(keyword arguement)作为输入。 其作用有两个: ## 1) 省略关键字参数的默认值 请看下面的简单例子

def func(arg1, arg2, *, arg3, arg4):
    print('This is arg1: ' + arg1)
    print('This is arg2: ' + arg2)
    print('This is arg3: ' + arg3)
    print('This is arg4: ' + arg4)

func('arg1', 'arg2', 'arg3', 'arg4')

返回

Exception has occurred: TypeError
func() takes 2 positional arguments but 4 were given

这里是说func只需要2个位置参数(position arguements), 但是接到了4个。修改代码如下:

def func(arg1, arg2, *, arg3, arg4):
    print('This is arg1: ' + arg1)
    print('This is arg2: ' + arg2)
    print('This is arg3: ' + arg3)
    print('This is arg4: ' + arg4)

func('arg1', 'arg2', arg3='arg3', arg4='arg4')

返回

This is arg1: arg1
This is arg2: arg2
This is arg3: arg3
This is arg4: arg4

再来看看arg3， arg4 有没有关键字参数的打包解包(pack-unpack)性质

def func(arg1, arg2, *, arg3, arg4):
    print('This is arg1: ' + arg1)
    print('This is arg2: ' + arg2)
    print('This is arg3: ' + arg3)
    print('This is arg4: ' + arg4)

kwarg = {'arg3':'arg3', 'arg4':'arg4'}
func('arg1', 'arg2', **kwarg)

返回

This is arg1: arg1
This is arg2: arg2
This is arg3: arg3
This is arg4: arg4

证明 arg3， arg4 确实是可以不赋默认值的关键字参数。下面来看第二个作用。

2) 避免 位置参数 和 关键字参数混用

有时候我们输入位置参数时， 会不小心多打， 导致关键字参数被输入错。

def func(arg1, arg2, arg3, arg4=False):
    print('This is arg1: ' + arg1)
    print('This is arg2: ' + arg2)
    print('This is arg3: ' + arg3)
    print('This is arg4: ' + arg4)

func('arg1', 'arg2', 'arg3', 'arg3')

返回

This is arg1: arg1
This is arg2: arg2
This is arg3: arg3
This is arg4: arg3

可以看到， 由于我们多输入了一个 'arg3', 导致arg4接到了错误参数。修改代码如下：

def func(arg1, arg2, arg3, *, arg4=False):
    print('This is arg1: ' + arg1)
    print('This is arg2: ' + arg2)
    print('This is arg3: ' + arg3)
    print('This is arg4: ' + arg4)

func('arg1', 'arg2', 'arg3', 'arg3')

返回

func() takes 3 positional arguments but 4 were given

有效防止了我们输入错误。

回到刚开始python threading 多线程库的问题,

class Thread:

    def __init__(self, group=None, target=None, name=None,
                 args=(), kwargs=None, *, daemon=None):

这里声明了 daemon 一定要用关键字参数输入, 说明开发者认为这个参数十分重要。事实上， 这个daemon意思是守护线程的意思，和阻塞线程正好是两个相反的逻辑顺序。 本文不做深入讲解， 读者只需要知道这里daemon设置为True， False， 或者保持默认参数，会导致这个线程出现三个完全不同的运行状态， 因而要使用*来提醒用户避免输错。 ## 参考
